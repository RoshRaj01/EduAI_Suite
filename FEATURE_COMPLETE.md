## 📊 Database Schema

```sql
CREATE TABLE lessons (
    id INTEGER PRIMARY KEY,
    course_id INTEGER NOT NULL,          -- Which course this lesson is for
    title VARCHAR,                       -- Lesson title
    topic VARCHAR NOT NULL,              -- Topic (e.g., "Recursion")
    syllabus_context TEXT,               -- Optional context from teacher
    lecture_flow TEXT,                   -- Generated by Groq
    examples TEXT,                       -- Generated by Groq
    activities TEXT,                     -- Generated by Groq
    quiz_questions TEXT,                 -- Generated by Groq
    created_by INTEGER NOT NULL,         -- Teacher who created it
    posted_at TIMESTAMP,                 -- When published (null = draft)
    created_at TIMESTAMP,                -- Creation time
    updated_at TIMESTAMP,                -- Last update time
    FOREIGN KEY (course_id) REFERENCES courses(id),
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

## 🎓 How It Works Under the Hood

### Lesson Generation Process

```
1. User enters: Topic = "Recursion"
2. Frontend sends: POST /lessons/generate
3. Backend receives request
4. Calls: GroqService.generate_lesson_plan(topic="Recursion")
5. Groq returns:
   "LECTURE FLOW:
    - Introduction (5m)
    - Core Concepts (15m)
    ...
    EXAMPLES:
    - Example 1: Fibonacci
    ...
    ACTIVITIES:
    ...
    QUIZ QUESTIONS:
    ..."
6. Service parses response into 4 sections
7. Returns JSON with structured content
8. Frontend displays in chat UI
9. Teacher reviews and clicks "Post"
10. Frontend calls: POST /lessons + POST /lessons/:id/post
11. Backend saves to database, sets posted_at timestamp
12. Lesson now visible to students
```

### Student Visibility

```
1. Student loads dashboard
2. Frontend calls: GET /lessons?posted_only=true
3. Backend returns:
   [
     {id: 1, title: "Recursion", topic: "Recursion", posted_at: "2026-04-22T10:30:00Z"},
     {id: 2, title: "Photosynthesis", topic: "Biology", posted_at: "2026-04-21T15:00:00Z"},
     ...
   ]
4. Frontend renders "New Lesson Plans" widget
5. Each lesson shows: title, topic, posted_date
6. User can click for details (future enhancement)
```
